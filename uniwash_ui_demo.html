🔥 Setup Firebase
Panduan integrasi Firebase untuk UniWash
Buka Firebase Console Dokumentasi
Mengapa Firebase? Gratis sampai 50k reads/hari, real-time by default, auth bawaan, dan tidak butuh backend server — cocok untuk skala awal UniWash (target 50–200 user).
1 Install Firebase SDK
<!-- Tambahkan di <head> file HTML kamu --> <script type="module"> import { initializeApp } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js"; import { getFirestore } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js"; import { getAuth } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js"; const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "uniwash.firebaseapp.com", projectId: "uniwash", storageBucket: "uniwash.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" }; const app = initializeApp(firebaseConfig); const db = getFirestore(app); const auth = getAuth(app); </script>
2 Struktur Firestore yang Direkomendasikan
// Koleksi utama UniWash: orders/ ← semua order {orderId}/ userId, kurirId, mitraid status: "antri" | "dijemput" | "dicuci" | "dikeringkan" | "diantar" | "selesai" service: "reguler" | "ekspres" | "bundel" weightKg, totalPrice pickupAddress, pickupTime createdAt, updatedAt users/ ← pelanggan {userId}/ name, email, phone kost: { name, address, room } activeBundle: { quota, used, expiresAt } couriers/ ← kurir freelance {kurirId}/ name, phone, status: "aktif"|"standby"|"jemput" totalTrips, rating partners/ ← mitra laundry {partnerId}/ name, address, capacityKgPerDay, commissionPct
3 Contoh: Simpan Order Baru ke Firestore
import { collection, addDoc, serverTimestamp } from "firebase/firestore"; async function createOrder(orderData) { try { const docRef = await addDoc(collection(db, "orders"), { ...orderData, status: "antri", createdAt: serverTimestamp(), updatedAt: serverTimestamp() }); console.log("Order tersimpan, ID:", docRef.id); return docRef.id; } catch (e) { console.error("Gagal simpan order:", e); } }
4 Real-time Listener — Status Order Langsung Update
import { doc, onSnapshot } from "firebase/firestore"; // Di halaman tracking pelanggan: function listenOrder(orderId) { const unsub = onSnapshot(doc(db, "orders", orderId), (snap) => { if (snap.exists()) { const order = snap.data(); updateTrackingUI(order.status); // update UI langsung tanpa reload } }); return unsub; // panggil unsub() untuk stop listening }
5 Security Rules Firestore
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Order: user hanya bisa baca order miliknya sendiri match /orders/{orderId} { allow read: if request.auth != null && resource.data.userId == request.auth.uid; allow write: if request.auth != null; } // Admin bisa baca semua match /{document=**} { allow read, write: if request.auth.token.admin == true; } } }
